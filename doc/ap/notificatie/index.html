Dit document beschrijft een applicatieprofiel voor de Generieke Notificatiecomponent, in dit geval Notificatie AP. Een applicatieprofiel is een specificatie die bestaande termen hergebruikt van verschillende basis-standaarden en daar meer specificiteit aan toevoegt. Daarbij wordt het gebruik van de termen toegelicht voor de uitwisseling van Vlaamse overheidsinformatie. Het AP beantwoord de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

1. Licentie

Deze specificatie van Informatie Vlaanderen is gepubliceerd onder de "Modellicentie Gratis Hergebruik - v1.0".

2. Conformiteit

Een applicatieprofiel is een specificatie voor gegevensuitwisseling dat bijkomende beperkingen introduceert voor het toepassen van vocabularia. Dergelijke bijkomende beperkingen kunnen de volgende elementen bevatten:

  • Verfijning van de terminologie (definitie van klassen en eigenschappen) consistent met de semantiek uit de betreffende specificaties met een welbepaald gebruik als doel;
  • Het bepalen van kardinaliteiten voor eigenschappen.
  • Externe terminologie (klassen en eigenschappen) gebruikt voor nieuwe/extra termen die niet in de bestaande vocabularia voorkomen.
  • Het gebruik van codelijsten.

Om conform te zijn met dit applicatieprofiel, MOET een implementatie:

  • Voor elke klasse steeds de eigenschappen bevatten die als minimum kardinaliteit 1 hebben.
  • Niet meer dan 1 instantie bevatten van eigenschappen die 1 als maximum kardinaliteit hebben.
  • De terminologie (klassen en eigenschappen) gebruiken op een manier die consistent is met haar semantiek (definitie, gebruik, domein en bereik).
  • Geen terminologie van andere gecontroleerde vocabularia gebruiken dan diegene gedefinieerd in dit applicatieprofiel.
  • Mag uitgebreid worden met klassen en eigenschappen van andere datamodellen (vocabularia) die niet overlappen met terminologie uit dit applicatieprofiel.

3. Overzicht

In dit document wordt correct gebruik van de volgende entiteiten toegelicht:

In dit document worden de volgende datatypes toegelicht:
| ContactInfo | SleutelWaardePaar |

GeregistreerdPersoon

Persoon waarvan de gegevens zijn ingeschreven in een register.

Doorgaans is dit register een bevolkingsregister maar het kan bv ook een kiesregister zijn. De ingeschreven gegevens hebben betrekking op de identeit (vb Naam en Voornaam) en de Verblijfplaats vd Persoon en op belangrijke levensgebeurtenissen zoals Geboorte, Huwelijk, Overlijden etc. Deze gegevens worden typisch geregistreerd door de overheid, ze bieden de ingeschreven Persoon wettelijke bescherming en laten de overheid toe om basisstatistieken op te stellen over zijn bevolking.

Voor deze entiteit zijn de volgende attributen gedefinieerd: isTeBereikenOp.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
isTeBereikenOp ContactInfo 0 .. 1 Informatie zoals email, telefoon die toelaat de Persoon te contacteren.
Voorbeeld JSON-LD: 
"@id": "https://data.vlaanderen.be/id/geregistreerdepersoon/00000014182",
"@type": "GeregistreerdPersoon",
"isTeBereikenOp": {
  "email": ["jan.jansens@gmail.com", "j.jansens@outlook.com"],
  "telefoon": ["0488010101", "0488020202"]
}

GeregistreerdeOrganisatie

Organisatie met een juridisch statuut vastgelegd door registratie. Vergelijk met Formele Organisatie waarbij dit statuut ook op een andere manier verkregen kan zijn.

Voor deze entiteit zijn de volgende attributen gedefinieerd: isTeBereikenOp.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
isTeBereikenOp ContactInfo 0 .. 1 Informatie zoals email, telefoon die toelaat de Organisatie te contacteren.
Voorbeeld JSON-LD: 
"@id": "https://data.vlaanderen.be/id/geregistreerdeorganisatie/1234",
"@type": "GeregistreerdeOrganisatie",
"isTeBereikenOp": {
  "email": ["info@examplenv.com"],
  "telefoon": ["02010101", "0488020202"]
}

Notificatiebericht

Een Notificatiebericht van een afzender naar een bestemmeling.

Voor deze entiteit zijn de volgende attributen gedefinieerd: url, vervalt, inhoud, creatie, titel, type, bron, bestemmeling, afzender.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
url URI 0 .. 1 Een url voor het Notificatiebericht. De url kan gebruikt worden om een doorverwijzing mee te geven naar een resource waarop het Notificatiebericht betrekking heeft.
vervalt DateTime 1 .. 1 De datum waarop een Notificatiebericht vervalt.
inhoud SleutelWaardePaar 0 .. * De inhoud van een Notificatiebericht. De inhoudt van een Notificatiebericht wordt gevormd door een aantal key-value pairs.
creatie DateTime 1 .. 1 De datum waarop het Notificatiebericht gecreëerd werd.
titel String 1 .. 1 Een titel voor het Notificatiebericht. De titel kan in praktijk automatisch worden samengesteld aan de hand van een aantal andere beschikbare velden.
type NotificatieType 1 .. 1 Het type van het Notificatiebericht.
bron PubliekeOrganisatie 1 .. 1 Duidt de bron aan van een Notificatiebericht.
bestemmeling Agent 1 .. * De bestemmeling of ontvanger van het notificatiebericht.
afzender Merk 1 .. 1 De afzender van het Notificatiebericht.
Voorbeeld JSON-LD: 
"@id": "https://data.vlaanderen.be/id/notificatiebericht/201707202351",
"@type": "Notificatiebericht",
"vervalt": "2017-07-20 00:00:00.000",
"creatie": "2017-07-10 00:00:00.000",
"titel": "Notificatie 10/07/2017 - Wijziging dossier",
"type": "WijzigingDossier",
"bron": {
  { 
  "@id": "https://wegwijs.vlaanderen.be/#/organisations/b078dfb1-4a17-4bfe-9342-1adf516c7587",

  "voorkeursNaam": "Administratieve diensten van de Vlaamse overheid"

    "@type": "GeregistreerdPersoon"
  }
],
"afzender": {

Notificatievoorkeur

De klasse om voorkeuren met betrekking tot notificaties te beschrijven.

Voor deze entiteit zijn de volgende attributen gedefinieerd: frequentie, notificatiekanaal, permissie.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
frequentie Frequentie 0 .. 1 De frequentie waarmee genotificeerd wordt.
notificatiekanaal Notificatiekanaal 1 .. 1 Het notificatiekanaal duidt het technisch medium aan waarlangs het Notificatiebericht wordt verzonden. Wordt in de context van Notificatievoorkeur gebruikt om aan te duiden voor welk notificatiekanaal de voorkeur geldt.
permissie PermissieType 1 .. 1 De permissie die werd toegekend door de bestemmeling om al dan niet notificaties te ontvangen van een bepaalde afzender met een bepaalde frequentie.
Voorbeeld JSON-LD: 
"@type": "Notificatievoorkeur",
"notificatiekanaal": "Email",
"permissie": "Toegelaten"

PersoonlijkProfielDocument

Een persoonlijk profiel document,voorgesteld in RDF.

Voor deze entiteit zijn de volgende attributen gedefinieerd: primairOnderwerp, notificatievoorkeur, maker.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
primairOnderwerp Agent 1 .. 1 Het primair onderwerp van dit profiel. In de meeste gevallen zal het primairOnderwerp gelijk zijn aan de maker. Dit zou kunnen verschillen in de context van profielen voor bedrijven (buiten de scope van dit applicatieprofiel), waar een werknemer een profiel aanmaakt voor de organisatie.
notificatievoorkeur Notificatievoorkeur 1 .. * De notificatievoorkeur van de entiteit.
maker Agent 1 .. 1 De Agent die het profiel heeft gecreëerd.
Voorbeeld JSON-LD:

"@type": "PersoonlijkProfielDocument",
"primairOnderwerp": {

  "@type": "GeregistreerdPersoon"
},
"notificatievoorkeur": {
  "@type": "Notificatievoorkeur",
  "notificatiekanaal": "Email",
  "permissie": "Toegelaten"
},
"maker": {
  "@id": "https://data.vlaanderen.be/id/geregistreerdepersoon/00000014182",

Status

De Status van het Notificatiebericht voor een bepaalde bestemmeling.

Voor deze entiteit zijn de volgende attributen gedefinieerd: datumVerzending, statusNotificatie, notificatiekanaal, datumGelezen, bestemmeling.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
datumVerzending DateTime 1 .. 1 De datum waarop het Notificatiebericht werd verzonden naar de bestemmeling.
statusNotificatie StatusNotificatie 1 .. 1 De status van een notificatie voor een bepaalde bestemmeling.
notificatiekanaal Notificiatiekanaal 1 .. * Het notificatiekanaal duidt het technisch medium aan waarlangs het Notificatiebericht wordt verzonden.
datumGelezen DateTime 1 .. 1 De datum waarop het Notificatiebericht werd gelezen door de bestemmeling.
notificatiebericht Notificatiebericht 0 .. * Het Notificatiebericht waarop deze Status betrekking heeft.
bestemmeling Agent 1 .. * De bestemmeling of ontvanger van het notificatiebericht.
Voorbeeld JSON-LD: 
"@type": "Status",
"datumVerzending": "2017-07-10 09:00:00.000",
"statusNotificatie": "Verstuurd",
"notificatiekanaal": ["Email", "Sms"],
"datumGelezen": "2017-07-10 11:53:12.453",
"notificatiebericht": "https://data.vlaanderen.be/id/notificatiebericht/201707202351",
"bestemmeling": "https://data.vlaanderen.be/id/geregistreerdepersoon/1234"

SleutelWaardePaar

Een open data structuur dat uitbreiding toelaat aan de hand van een sleutel en een waarde.

Voor dit datatype zijn de volgende attributen gedefinieerd: label, waarde.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
label String 1 .. 1 Het label is de sleutel van het sleutel/waarde paar.
waarde String 1 .. 1 De waarde van het sleutel/waarde paar.
Voorbeeld JSON-LD: 
"@id": "https://data.vlaanderen.be/id/notificatiebericht/201707202351",
"@type": "Notificatiebericht",
"inhoud": {
  "label": "roepnaam",
  "waarde": "Jef"
}
