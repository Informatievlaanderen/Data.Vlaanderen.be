Dit document beschrijft een applicatieprofiel voor Dienst, in dit geval Dienstverlening Transactiemodel AP. Dit applicatieprofiel dient voor het beschrijven van transactionele dienstverlening; het beschrijven van publieke diensten die in praktijk worden afgenomen door burgers of ondernemingen. In de basis beschrijft dit applicatieprofiel de publieke dienstverlening en de personen en (publieke) organisaties die erin participeren. Verder wordt toegelicht hoe ook de status van de transactionele dienstverlening kan worden gespecifieerd, alsook hoe de concepten bewijs, output en kanaal ermee verband houden. Een applicatieprofiel is een specificatie die bestaande termen hergebruikt van verschillende basis-standaarden en daar meer specificiteit aan toevoegt. Daarbij wordt het gebruik van de termen toegelicht voor de uitwisseling van Vlaamse overheidsinformatie. Het AP beantwoord de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

1. Licentie en aansprakelijkheid

Deze specificatie van Informatie Vlaanderen is gepubliceerd onder de "Modellicentie Gratis Hergebruik - v1.0".

2. Conformiteit

Een applicatieprofiel is een specificatie voor gegevensuitwisseling dat bijkomende beperkingen introduceert voor het toepassen van vocabularia. Dergelijke bijkomende beperkingen kunnen de volgende elementen bevatten:

  • Verfijning van de terminologie (definitie van klassen en eigenschappen) consistent met de semantiek uit de betreffende specificaties met een welbepaald gebruik als doel;
  • Het bepalen van kardinaliteiten voor eigenschappen.
  • Externe terminologie (klassen en eigenschappen) gebruikt voor nieuwe/extra termen die niet in de bestaande vocabularia voorkomen.
  • Het gebruik van codelijsten.

Om conform te zijn met dit applicatieprofiel, MOET een implementatie:

  • Voor elke klasse steeds de eigenschappen bevatten die als minimum kardinaliteit 1 hebben.
  • Niet meer dan 1 instantie bevatten van eigenschappen die 1 als maximum kardinaliteit hebben.
  • De terminologie (klassen en eigenschappen) gebruiken op een manier die consistent is met haar semantiek (definitie, gebruik, domein en bereik).
  • Geen terminologie van andere gecontroleerde vocabularia gebruiken dan diegene gedefinieerd in dit applicatieprofiel.
  • Mag uitgebreid worden met klassen en eigenschappen van andere datamodellen (vocabularia) die niet overlappen met terminologie uit dit applicatieprofiel.

Bewijs

Elk stuk waarmee kan worden aangetoond dat men aan een CriteriumVereiste voldoet.

Voor deze entiteit zijn de volgende attributen gedefinieerd: beschrijving, documentatie, naam, taal, type.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
beschrijving String 0 .. 1 Beschrijving vh Bewijs.
documentatie URI 0 .. * Documentatie gerelateerd ah Bewiijs. Bvb template van een administratief document of aanvraagformulier of aanwijzingen voor het invullen ervan.
naam String 1 .. 1 Officiële naam vh Bewijs.
taal NAL_language 0 .. * Taal waarin het aangeboden Bewijs moet zijn gesteld. Link
type BewijsType 0 .. 1 Aard vh Bewijs.

Kanaal

Kanaal waarlangs een Agent kan interageren met de Publieke Dienstverlening.

Voor deze entiteit zijn de volgende attributen gedefinieerd: beschikbaarheid, openingsuren, type.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
beschikbaarheid OpeningsurenSpecificatie 0 .. 1 Periode waarin de Publieke Dienstverlening beschikbaar is via het Kanaal. Bv startdatum zonder einddatum voor dienstverlening die nog actief is of een aanduiding van het seizoen waarin de dienstverlening wordt aangeboden. Een ingevulde einddatum geeft aan wanneer de dienstverlening werd beëindigd. Begin- en einddatum volgens ISO8601 syntax.
openingsuren String 0 .. * Uren waarbinnen de Publieke Dienstverlening beschikbaar is via het Kanaal. Syntax volgens schema.org voor openingsuren. Bv als een dienstverlening telefonisch bereikbaar is tussen 8 en 20u maandag tot zaterdag and tussen 8 en 18u op zondag wordt dit gecodeerd als Ma-za 08:00-20:00, zo 08:00-18:00.
type KanaalType 0 .. 1 Aard vh Kanaal.

PubliekeDienstverlening

Een Publieke Dienstverlening is een geheel van verplichte of optioneel uitgevoerde of uitvoerbare acties door of in naam van een publieke organisatie. De dienstverlening is ten bate van een individu, een bedrijf, een andere publieke organisatie of groepen hiervan.

Voor deze entiteit zijn de volgende attributen gedefinieerd: beschrijving, contactinfo, kost, naam, sector, taal, type, executieStatus, verwerkingstijd, heeftParticiperende, produceert, wordtUitgevoerdDoor, isGerelateerdMet, vereistAlsInput, heeftStatus, teConsumerenVia.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
beschrijving String 0 .. 1 Beschrijving van de Publieke Dienstverlening.
contactinfo ContactInfo 0 .. 1 Informatie waarmee de Publieke Dienstverlening kan worden gecontacteerd. Verschillend van de informatie waarmee de Publieke Organisatie kan worden gecontacteerd.
kost Kost 0 .. 1 Officiële naam van de Publieke Dienstverlening.
naam String 1 .. 1 Officiële naam van de Publieke Dienstverlening.
sector NACE 0 .. * Sector of bedrijvigheid waarop de Publieke Dienstverlening betrekking heeft of waarvoor deze is bedoeld. Bv milieu, veiligheid, wonen...
taal NAL_language 0 .. * Taal waarin een Publieke Dienstverlening wordt aangeboden. Link
type COFOG 0 .. * Soort Publieke Dienstverlening. Bvb bescherming, gezondheid, recreatie, cultuu en religie, economische zaken.
executieStatus StatusVlaamseFase 0 .. * Soort Publieke Dienstverlening. Bvb bescherming, gezondheid, recreatie, cultuu en religie, economische zaken.
verwerkingstijd String 0 .. 1 Geschatte tijd nodig om de Publieke Dienstverlening af te leveren.
heeftParticiperende Participatie .. TODO
produceert Output 0 .. * Het werkelijke resultaat van de uitvoering vd Publieke Dienstverlening. Bvb een erkenning, financieel voordeel, code, fysiek object
wordtUitgevoerdDoor Agent 0 .. * Agent die de Publieke Dienstverlening verstrekt. Staat los van de rol die de bevoegde instantie speelt, kan een externe aannemer zijn.
isGerelateerdMet PubliekeDienstverlening 1 .. 1 Andere Publieke Dienstverlening waarmee de Publieke Dienstverlening gerelateerd is.
vereistAlsInput Bewijs 0 .. * Bewijsstuk dat moeten voorgelegd worden om de Publieke Dienstverlening te kunnen gebruiken. Als de bewijsstukken afhangen het kanaal waarmee met de Publieke Dienstverlening wordt geïnterageerd gebruiken we de associatie Kanaal-vereistAlsInput-Bewijs.
heeftStatus Status 0 .. * Link tussen Publieke Dienstverlening en Status.
teConsumerenVia Kanaal 0 .. * Kanaal waarlangs een Agent interageert met een Publieke Dienstverlening. Bv kantoor, telefoon, website

Kost

Deel vd kost verbonden aan de uitvoering van de Dienstverlening te betalen door de gebruiker.

Voor dit datatype zijn de volgende attributen gedefinieerd: beschrijving, valuta, waarde.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
beschrijving String 0 .. 1 Beschrijving van de Kost.
valuta NAL_currency 0 .. 1 Munteenheid waarin de Kost is gedefinieerd. Link
waarde Double 0 .. 1 Omvang van de Kost, uitgedrukt in de opgegeven munteenheid.

Status

De status van het dossier.

Voor dit datatype zijn de volgende attributen gedefinieerd: actie, actieNodig, isPubliek, isVertrouwelijk, statusDetail1, statusVlaamseCode, statusVlaamseFase, streefdatum, statusdatum, statusDetail2.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
actie 0 .. 1 Omschrijving van de uit te voeren actie.
actieNodig boolean 0 .. 1 Aanduiding of de huidige status impliceert dat er actie van burger of onderneming nodig is.
isPubliek 0 .. 1 Aanduiding of de informatie met betrekking tot de dienstverlening publiek is.
isVertrouwelijk 0 .. 1 Aanduiding of de informatie met betrekking tot de dienstverlening vertruwelijk is.
statusDetail1 0 .. 1 Dit is de (originele) Status-definitie of -vermelding zoals geregistreerd in de bron. Indien ingevuld wordt deze informatie meegegeven aan de burger of onderneming.
statusVlaamseCode StatusVlaamseCode 0 .. 1 De Vlaamse status - Code. Verplicht in geval van een dossier van het type DossierStatus. Link
statusVlaamseFase StatusVlaamseFase 0 .. 1 De Vlaamse status - Fase. Verplicht in geval van een dossier van het type DossierStatus. Link
streefdatum 0 .. 1 De streefdatum voor het beëindigen van de huidige status.
statusdatum 0 .. 1 De datum van de Status. Bijvoorbeeld de status wijzigingsdatum.
statusDetail2 0 .. 1 Interne status die kan worden meegegeven en die niet wordt getoond aan de burger of onderneming. Deze interne status kan meegegeven worden omwille van verschillende redenen: inzage door eigen ambtenaren, inzage door andere gemachtigde ambtenaren, of voor B.I.- en/of DWH-doeleinden.
