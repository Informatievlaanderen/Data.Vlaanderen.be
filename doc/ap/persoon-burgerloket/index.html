Dit document beschrijft een applicatieprofiel voor Persoon, in dit geval Persoon Burgerloket AP. Een applicatieprofiel is een specificatie die bestaande termen hergebruikt van verschillende basis-standaarden en daar meer specificiteit aan toevoegt. Daarbij wordt het gebruik van de termen toegelicht voor de uitwisseling van Vlaamse overheidsinformatie. Het AP beantwoord de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

1. Licentie en aansprakelijkheid

Deze specificatie van Informatie Vlaanderen is gepubliceerd onder de "Modellicentie Gratis Hergebruik - v1.0".

2. Conformiteit

Een applicatieprofiel is een specificatie voor gegevensuitwisseling dat bijkomende beperkingen introduceert voor het toepassen van vocabularia. Dergelijke bijkomende beperkingen kunnen de volgende elementen bevatten:

  • Verfijning van de terminologie (definitie van klassen en eigenschappen) consistent met de semantiek uit de betreffende specificaties met een welbepaald gebruik als doel;
  • Het bepalen van kardinaliteiten voor eigenschappen.
  • Externe terminologie (klassen en eigenschappen) gebruikt voor nieuwe/extra termen die niet in de bestaande vocabularia voorkomen.
  • Het gebruik van codelijsten.

Om conform te zijn met dit applicatieprofiel, MOET een implementatie:

  • Voor elke klasse steeds de eigenschappen bevatten die als minimum kardinaliteit 1 hebben.
  • Niet meer dan 1 instantie bevatten van eigenschappen die 1 als maximum kardinaliteit hebben.
  • De terminologie (klassen en eigenschappen) gebruiken op een manier die consistent is met haar semantiek (definitie, gebruik, domein en bereik).
  • Geen terminologie van andere gecontroleerde vocabularia gebruiken dan diegene gedefinieerd in dit applicatieprofiel.
  • Mag uitgebreid worden met klassen en eigenschappen van andere datamodellen (vocabularia) die niet overlappen met terminologie uit dit applicatieprofiel.

Afstamming

Afstamming is de dalende lijn van bloedverwantschap tussen verschillende generaties.

De afstamming kan ook niet-biologisch zijn, bv adoptie. Afstamming kan ook in opgaande lijn worden bekeken. Het begrip is enger dan verwantschap dat oa ook huwelijk inhoudt.

Voor deze entiteit zijn de volgende attributen gedefinieerd: afstammingsType, datumVanAfstamming.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
afstammingsType Afstammingstype 1 .. 1 Aard van de afstamming. Bv. geadopteerd, kind uit huwelijk, erkend door de vader etc.
datumVanAfstamming GetypeerdeString 0 .. 1 De datum waarop de afstamming wordt vastgesteld.
Voorbeeld JSON-LD: 
"@id": "https://data.vlaanderen.be/id/persoon/00000014182",
"@type": [ "GeregistreerdPersoon", "Inwoner" ],
"heeftRelatieMet": [
  {
    "@id": "https://data.vlaanderen.be/id/persoon/48000327561",
    "@type": "Afstamming",
    "datumVanAfstamming": "2000-00-00",
    "typeAfstamming": {
      "@id": "https://data.vlaanderen.be/id/typeAfstamming/10",
      "naam": "Kind"
    }
  }
]

BurgerlijkeStaat

Burgerrechtelijke toestand van een Persoon.

Slaat op huwelijk, partnerregistratie, afstamming, voogdij etc. Is maw de toestand van bepaalde verhoudingen tussen personen.

Voor deze entiteit zijn de volgende attributen gedefinieerd: type.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
type BurgerlijkeStaatType 1 .. 1 Aard van de Burgerlijke Staat.
Voorbeeld JSON-LD: 
"@id": "https://data.vlaanderen.be/id/persoon/00000014182",
"@type": [ "GeregistreerdPersoon", "Inwoner" ],
"heeftBurgerlijkeStaat": {
    "type": {
      "@id": "http://data.vlaanderen.be/id/burgerlijkestaattype/10",
      "naam": "Ongehuwd"
    }

GeregistreerdPersoon

Persoon waarvan de gegevens zijn ingeschreven in een register.

Doorgaans is dit register een bevolkingsregister maar het kan bv ook een kiesregister zijn. De ingeschreven gegevens hebben betrekking op de identeit (vb Naam en Voornaam) en de Verblijfplaats vd Persoon en op belangrijke levensgebeurtenissen zoals Geboorte, Huwelijk, Overlijden etc. Deze gegevens worden typisch geregistreerd door de overheid, ze bieden de ingeschreven Persoon wettelijke bescherming en laten de overheid toe om basisstatistieken op te stellen over zijn bevolking.

Voor deze entiteit zijn de volgende attributen gedefinieerd: registratie.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
registratie Identificator 1 .. 1 Identificatiecode van de persoon in het register.
Voorbeeld JSON-LD: 
"@id": "https://data.vlaanderen.be/id/persoon/00000014182",
"@type": "GeregistreerdPersoon",
"registratie": {
  "identificator": "00000014182"
}

Gezin

Persoon

Natuurlijk persoon.

In de rechtspraak betreft het een persoon (in de wettelijke betekenis, ttz met eigen rechtspersoonlijkheid) van de menselijke soort, ttz een fysiek persoon. Tegenhanger is de rechtspersoon, een juridische constructie die een private of publieke Organisatie dezelfde rechtspersoonlijkheid geeft als een natuurlijk persoon (kan bv ook schulden hebben, contracten afsluiten, aangeklaagd worden etc).

Voor deze entiteit zijn de volgende attributen gedefinieerd: achternaam, alternatieveNaam, geboortenaam, gebruikteVoornaam, geslacht, isTeBereikenOp, juridischStatuut, patroniem, volledigeNaam, voornaam, heeftBurgerlijkeStaat, heeftOverlijden, heeftRelatieMet, inwonerschap, isHoofdVan, isLidVan, heeftGeboorte, heeftNationaliteit.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
achternaam String 1 .. 1 Gedeelte van de Volledige Naam vd Persoon ontvangen van de vorige generatie. Ook wel familienaam genoemd omdat de Achternaam een familiale verwantschap aanduidt.
alternatieveNaam String 0 .. * Alternatief voor de Volledige Naam vd Persoon. Bv pseudoniem, titel etc.
geboortenaam String 0 .. 1 Volledige Naam vd Persoon bij geboorte. De namen van een Persoon kunnen id loop vd tijd wijzigen, bv kan de Achternaam wijzigen door huwelijk. De oorspronkelijke naam wordt echter dikwijls ook nog gebruikt.
gebruikteVoornaam String 1 .. 1 Belangrijkste vd voornamen ve Persoon.
geslacht Geslacht 1 .. 1 Het feit of men een man of een vrouw is.
isTeBereikenOp ContactInfo 0 .. 1 Informatie zoals email, telefoon die toelaat de Persoon te contacteren.
juridischStatuut JuridischStatuut 0 .. * Juridische statuut van de persoon. Voor een minderjarig persoon: het feit dat hij ontvoogd werd. Voor een meerderjarig persoon: de informatie met betrekking tot de beslissing(en) waarbij beschermende maatregelen worden opgelegd.
patroniem String 0 .. 1 Achternaam gebaseerd op de voornaam vd vader ve Persoon. Bv in IJsland: Magnusdottir (dochter van Magnus).
volledigeNaam String 1 .. 1 De volledige naam vd Persoon, doorgaans de combinatie van voornamen en achternaam.
voornaam String 1 .. * Naam die een kind bij geboorte wordt gegeven. Onderscheidt het kind van de andere kinderen in de familie.
heeftBurgerlijkeStaat BurgerlijkeStaat 0 .. 1 Burgerlijke Staat vd Persoon.
heeftOverlijden Overlijden 0 .. 1 Verwijst naar de overlijdensgegevens vd Persoon.
heeftRelatieMet Persoonsrelatie .. Persoon waarmee de Persoon gerelateerd is.
inwonerschap Inwonerschap .. Jurisdictie waarbinnen het Inwonerschap vd Persoon is gedefinieerd.
isHoofdVan Gezin 0 .. 1 Persoon die standaard het Gezin vertegenwoordigt.
isLidVan Gezin 1 .. 1 Persoon die tot een Gezin behoort.
heeftGeboorte Geboorte 0 .. 1 Verwijst naar de geboortegegevens vd Persoon.
heeftNationaliteit Nationaliteit 0 .. * Nationaliteit vd Persoon.
Voorbeeld JSON-LD: 
"@id": "http://data.vlaanderen.be/id/persoon/84103118115",
"@type": "Persoon",
"achternaam": [ "Jansens" ],
"voornaam": [ "Jan" ],
"geslacht": {
  "@id": "http://data.vlaanderen.be/id/geslacht/1",
  "naam": "Mannelijk"
},
"heeftNationaliteit": [
  {
    "@id": "http://publications.europa.eu/resource/authority/country/BEL"
  }
