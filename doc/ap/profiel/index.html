Dit document beschrijft een applicatieprofiel voor OSLO²_applicatieprofiel_Persoon_Profiel, in dit geval profiel-ap. Een applicatieprofiel is een specificatie die bestaande termen hergebruikt van verschillende basis-standaarden en daar meer specificiteit aan toevoegt. Daarbij wordt het gebruik van de termen toegelicht voor de uitwisseling van Vlaamse overheidsinformatie. Het AP beantwoord de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

1. Licentie en aansprakelijkheid

Deze specificatie van Informatie Vlaanderen is gepubliceerd onder de "Modellicentie Gratis Hergebruik - v1.0".

2. Conformiteit

Een applicatieprofiel is een specificatie voor gegevensuitwisseling dat bijkomende beperkingen introduceert voor het toepassen van vocabularia. Dergelijke bijkomende beperkingen kunnen de volgende elementen bevatten:

  • Verfijning van de terminologie (definitie van klassen en eigenschappen) consistent met de semantiek uit de betreffende specificaties met een welbepaald gebruik als doel;
  • Het bepalen van kardinaliteiten voor eigenschappen.
  • Externe terminologie (klassen en eigenschappen) gebruikt voor nieuwe/extra termen die niet in de bestaande vocabularia voorkomen.
  • Het gebruik van codelijsten.

Om conform te zijn met dit applicatieprofiel, MOET een implementatie:

  • Voor elke klasse steeds de eigenschappen bevatten die als minimum kardinaliteit 1 hebben.
  • Niet meer dan 1 instantie bevatten van eigenschappen die 1 als maximum kardinaliteit hebben.
  • De terminologie (klassen en eigenschappen) gebruiken op een manier die consistent is met haar semantiek (definitie, gebruik, domein en bereik).
  • Geen terminologie van andere gecontroleerde vocabularia gebruiken dan diegene gedefinieerd in dit applicatieprofiel.
  • Mag uitgebreid worden met klassen en eigenschappen van andere datamodellen (vocabularia) die niet overlappen met terminologie uit dit applicatieprofiel.

BurgerlijkeStaat

Beschrijft de burgerrechtelijke toestand van een Persoon.

Voor deze entiteit zijn de volgende attributen gedefinieerd: type.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
type BurgerlijkeStaatType 1 .. 1 De aard van de Burgerlijke Staat

GeregistreerdPersoon

Persoon waarvan de gegevens zijn ingeschreven in een register.

Voor deze entiteit zijn de volgende attributen gedefinieerd: registratie.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
registratie Identificator 1 .. 1 Identificatiecode van de Persoon in het register. In praktijk het Rijksregisternummer of INSZ.
Voorbeeld JSON-LD: 
"@id": "https://data.vlaanderen.be/id/persoon/00000014182",
"@type": "GeregistreerdPersoon",
"registratie": {
  "identificator": "00000014182"
}

Persoon

De Persoon die het onderwerp vormt van het Profiel.

Voor deze entiteit zijn de volgende attributen gedefinieerd: achternaam, adellijkeTitel, alternatieveNaam, bankgegevens, geboortenaam, gebruikteVoornaam, geslacht, heeftTaal, img, isTeBereikenOp, patroniem, volledigeNaam, voornaam, heeftNationaliteit, heeftBurgerlijkeStaat, heeftVerblijfplaats.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
achternaam String 1 .. * De familienaam van de Persoon.
adellijkeTitel String 0 .. 1 De adelijke titel van de Persoon.
alternatieveNaam String 0 .. * De alternatieve naam van de Persoon. De alternatieve naam kan gebruikt worden om bijvoorbeeld een bijnaam te capteren of een in praktijk gebruikte variant op de voornaam.
bankgegevens Bankgegevens 0 .. 1 De bankgegevens van een persoon.
geboortenaam String 0 .. 1 De naam van de Persoon zoals toegewezen bij de geboorte.
gebruikteVoornaam String 1 .. 1 De voornaam die in praktijk gebruikt wordt. Dit attribuut is relevant indien de Persoon meerdere voornamen bezit, maar slechts één in praktijk wordt gebruikt om de persoon aan te spreken.
geslacht Geslacht 1 .. 1 Het geslacht van de Persoon of het geslacht waarmee de Persoon zich identificeert
heeftTaal Taal 0 .. * Specifieert de taal die mag gebruikt worden om het object te contacteren (bv. persoon of organisatie).
img Image 0 .. 1 Een afbeelding die gebruikt wordt om iets of iemand voor te stellen.
isTeBereikenOp ContactInfo 0 .. 1 Informatie zoals email, telefoon die toelaat de Persoon te contacteren.
patroniem String 0 .. 1 Een persoonsnaam ontleend aan de voornaam van de vader van de Persoon. Patroniemen zijn belangrijk in sommige landen. IJsland heeft bijvoorbeeld geen concept familienaam zoals we dit kennen in vele Europese landen. In Bulgarije en Rusland worden patroniemen dagelijks gebruikt, bijvoorbeeld Sergeyevich" (?????????) in "Mikhail Sergeyevich Gorbachev" (?????? ????????? ????????).
volledigeNaam String 1 .. 1 De volledige naam van de Persoon, omvat alle voor- en achternamen van een persoon.
voornaam String 1 .. * De gegeven naam van de Persoon.
heeftNationaliteit Nationaliteit 0 .. * Nationaliteit van de persoon.
heeftBurgerlijkeStaat BurgerlijkeStaat 0 .. 1 Burgerlijke Staat van de persoon.
heeftVerblijfplaats Verblijfplaats 0 .. * Plaats waar een persoon verblijft.
Voorbeeld JSON-LD: 
"@id": "https://data.vlaanderen.be/id/persoon/00000014182", 
"@type": "Persoon",
"voornaam": [ "Jan" ],
"achternaam": [ "Jansens" ],
"volledigeNaam": "Jan Jansens",
"geslacht": "Man",
"img": "http://example.org/profielfoto.jpeg"
"heeftTaal": [
  {
    "@id": "http://publications.europa.eu/resource/authority/language/NLD",
    "taal": "Nederlands",
    "type": "Moedertaal"
  }
],
"isTeBereikenOp": {
  "email": ["jan.jansens@gmail.com", "j.jansens@outlook.com"],
  "telefoon": ["0488010101", "0488020202"]
}

PersoonlijkProfielDocument

Een persoonlijk profiel document, voorgesteld in RDF.

Voor deze entiteit zijn de volgende attributen gedefinieerd: status, primairOnderwerp, maker.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
status ProfielStatus 1 .. 1 De status van het profiel.
primairOnderwerp Agent 1 .. 1 Het primair onderwerp van dit profiel. In de meeste gevallen zal het primairOnderwerp gelijk zijn aan de maker. Dit zou kunnen verschillen in de context van profielen voor bedrijven (buiten de scope van dit applicatieprofiel), waar een werknemer een profiel aanmaakt voor de organisatie.
maker Agent 1 .. 1 De Agent die het profiel heeft gecreëerd.
Voorbeeld JSON-LD: 
"@id": "http://data.vlaanderen.be/id/profiel/burgerloket/1234",
"@type": "PersoonlijkProfielDocument",
"status": "actief",
"maker": { "@id": "https://data.vlaanderen.be/id/persoon/00000014182" },
"primairOnderwerp": { 
  "@id": "https://data.vlaanderen.be/id/persoon/00000014182", 
  "@type": "Persoon",
}

Datatypes

Adresvoorstelling

Complex datatype voor het beschrijven van een adres.

Voor dit datatype zijn de volgende attributen gedefinieerd: administratieveEenheidNiveau1, administratieveEenheidNiveau2, adresgebied, busnummer, gemeentenaam, huisnummer, land, locatieaanduiding, locatienaam, postbus, postcode, postnaam, straatnaam, volledigAdres.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
administratieveEenheidNiveau1 String 0 .. 1 Hoogste Administratieve eenheid vh adres, doorgaans een land.
administratieveEenheidNiveau2 String 0 .. 1 De regio vh adres, doorgaans een provincie of deelstaat of gelijkaardig gebied dat typisch meerdere plaatsen omvat.
adresgebied String 0 .. 1 Naam ve geografisch gebied of plaats die een aantal adresseerbare objecten groepeert om deze te adresseren zonder dat het gebied of de plaats een administratieve eenheid is Bvb de naam vh gehucht waarin het adres ligt.
busnummer String 0 .. 1 Officieel toegekende alfanumerieke code die wordt toegevoegd aan het huisnummer om meerdere gebouweenheden, standplaatsen, ligplaatsen of percelen te onderscheiden die eenzelfde huisnummer hebben. Specialisatie van Adresvoorstelling:locatieaanduiding tbv Belgische adressen.
gemeentenaam String 0 .. 1 Gemeentenaam vh adres.
huisnummer String 0 .. 1 Alfanumerieke code officieel toegekend aan gebouweenheden, ligplaatsen, standplaatsen of percelen. Specialisatie van Adresvoorstelling:locatieaanduiding tbv Belgische adressen.
land String 0 .. 1 Land waarin het adres gelegen is.
locatieaanduiding String 0 .. 1 Alfanumerieke code die de adreslocator uniek identificeert binnen de straat, administratieve eenheid etc.
locatienaam String 0 .. 1 Naam of omschrijving vh het geografisch object dat de adreslocator aanduidt.
postbus String 0 .. 1 Postbusnummer in een postkantoor.
postcode String 0 .. 1 Code waarmee het geografisch gebied dat adressen voor postale doeleinden groepeert wordt aangeduid.
postnaam String 0 .. 1 Naam waarmee het geografisch gebied dat adressen voor postale doeleinden groepeert wordt aangeduid.
straatnaam String 0 .. 1 Straatnaam vh adres.
volledigAdres String 0 .. 1 Het complete adres in n string, al dan niet geformatteerd. Vermijdt fouten tgv het opsplitsen ve adres in zijn onderdelen. Geeft de voorgeschreven volgorde vd verschillende onderdelen weer.
Voorbeeld JSON-LD: 
"@id": "https://data.vlaanderen.be/id/persoon/00000014182",
"heeftVerblijfplaats": [
  {
    "@type": "Domicilie"
    "verblijfsadres": {
      "straatnaam": "Lubbeekstraat",
      "huisnummer": "20",
      "gemeentenaam": "Lubbeek",
      "land": "Belgi"
    }
  }        
]

Voor dit datatype zijn de volgende attributen gedefinieerd: bankrekeningnummer, swiftcode.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
bankrekeningnummer String 1 .. 1 Het rekeningummer van de Persoon.
swiftcode String 1 .. 1 De SWIFT code geassocieerd met het rekeningnummer.
Voorbeeld JSON-LD:
