Dit document beschrijft een applicatieprofiel voor Melding, in dit geval Melding Basis AP. Een applicatieprofiel is een specificatie die bestaande termen hergebruikt van verschillende basis-standaarden en daar meer specificiteit aan toevoegt. Daarbij wordt het gebruik van de termen toegelicht voor de uitwisseling van Vlaamse overheidsinformatie. Het AP beantwoord de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

1. Licentie en aansprakelijkheid

Deze specificatie van Informatie Vlaanderen is gepubliceerd onder de "Modellicentie Gratis Hergebruik - v1.0".

2. Conformiteit

Een applicatieprofiel is een specificatie voor gegevensuitwisseling dat bijkomende beperkingen introduceert voor het toepassen van vocabularia. Dergelijke bijkomende beperkingen kunnen de volgende elementen bevatten:

  • Verfijning van de terminologie (definitie van klassen en eigenschappen) consistent met de semantiek uit de betreffende specificaties met een welbepaald gebruik als doel;
  • Het bepalen van kardinaliteiten voor eigenschappen.
  • Externe terminologie (klassen en eigenschappen) gebruikt voor nieuwe/extra termen die niet in de bestaande vocabularia voorkomen.
  • Het gebruik van codelijsten.

Om conform te zijn met dit applicatieprofiel, MOET een implementatie:

  • Voor elke klasse steeds de eigenschappen bevatten die als minimum kardinaliteit 1 hebben.
  • Niet meer dan 1 instantie bevatten van eigenschappen die 1 als maximum kardinaliteit hebben.
  • De terminologie (klassen en eigenschappen) gebruiken op een manier die consistent is met haar semantiek (definitie, gebruik, domein en bereik).
  • Geen terminologie van andere gecontroleerde vocabularia gebruiken dan diegene gedefinieerd in dit applicatieprofiel.
  • Mag uitgebreid worden met klassen en eigenschappen van andere datamodellen (vocabularia) die niet overlappen met terminologie uit dit applicatieprofiel.

3. Overzicht

In dit document wordt correct gebruik van de volgende entiteiten toegelicht:
| Agent | Dataset | Eigenschap | Melding | Meldingsobject | Participatie | Status | TijdToestand |

In dit document worden de volgende datatypes toegelicht:
| Geometrie | MeldingBody | MeldingsobjectAttribuut |

Agent

Voor deze entiteit zijn de volgende attributen gedefinieerd: speelt rol.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
speelt rol Participatie 1 .. * De relatie tussen een Melding en een Agent die een rol speelt in de Melding.
Voorbeeld JSON-LD: 
"@id": "http://data.vlaanderen.be/id/melding/1234",
"@type": "Melding",
"heeftParticiperende": [
  {
    "@id": "https://data.vlaanderen.be/id/persoon/00000056789",
    "@type": "Persoon",
    "rol": "Indiener"
  }
]

Dataset

De Dataset klasse beschrijft die datasets waarop een melding betrekking kan hebben.

Voor deze entiteit zijn de volgende attributen gedefinieerd: attribuut, beschrijving, titel.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
beschrijving String 1 .. 1 Een beschrijving van de Dataset.
eigenschap Eigenschap 0 .. * Verwijst naar de Eigenschap die deel uitmaakt van de Dataset.
titel String 1 .. 1 De titel van een Dataset.
Voorbeeld JSON-LD: 
"@id": "http://data.vlaanderen.be/id/dataset/grb",
"@type": "Dataset",
"titel": "Grootschalig Referentie Bestand",
"beschrijving": "Het Grootschalig Referentie Bestand of Basiskaart Vlaanderen (GRB) is een 
geografisch informatiesysteem dat dient als topografische referentie voor Vlaanderen.",
"eigenschap": [
  "http://data.vlaanderen.be/id/eigenschap/grb/geometrie", 
  "http://data.vlaanderen.be/id/eigenschap/grb/adres", 
  "http://data.vlaanderen.be/id/eigenschap/grb/type"
]

Eigenschap

De klasse voor het beschrijven van de eigenschappen waaruit een dataset bestaat.

Voor deze entiteit zijn de volgende attributen gedefinieerd: bereik, codelijst, label, verplicht, regex, werd toegewezen aan.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
bereik Datatype 0 .. 1 Geeft de verwachte waarde aan van de Eigenschap in de vorm van een Datatype.
codelijst CodelijstWaarde 0 .. * Associeert de Eigenschap met mogelijk codelijst waarden die voor de Eigenschap gebruikt kunnen worden.
label String 1 .. 1 Een label voor de Eigenschap.
regex String 0 .. 1 Een regular expression die het verwachte patroon aangeeft voor de waarden van de Eigenschap.
verplicht Boolean 0 .. 1 True als de eigenschap verplicht mee te sturen is, anders false.
werd toegewezen aan Agent 0 .. 1 Het toewijzen van de Eigenschap aan een Agent (Publieke Organisatie) die verantwoordelijk is voor het behandelen van meldingen op deze Eigenschap. Dit attribuut wordt genegeerd indien werdToegewezenAan gedefinieerd is op niveau van CodelijstWaarde.
Voorbeeld JSON-LD: 
"@id": "http://data.vlaanderen.be/id/eigenschap/led/UITREIKINGSDATUM",
"label": "Uitreikingsdatum",
"werdToegewezenAan": "https://wegwijs.vlaanderen.be/#/organisations/5c63f9ff-177d-4f2b-9922-033745d81b6a",
"verplicht": false,
"datatype": "xsd:Date",
"regex": "^\d{4}\-(0?[1-9]|1[012])\-(0?[1-9]|[12][0-9]|3[01])$"

Melding

De klasse voor digitale annotaties voor het melden van feedback op data en reële wereld objecten.

Voor deze entiteit zijn de volgende attributen gedefinieerd: meldingsapplicatie, meldingsorganisatie, meldingstype, samenvatting, titel, url, heeftStatus, heeftParticiperende, heeftBody, heeftDoelwit.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
meldingsapplicatie Meldingsapplicatie 1 .. 1 De applicatie dat de Melding creëerde.
meldingsorganisatie Organisatie 0 .. 1 De organisatie die verantwoordelijk is voor de applicatie die werd gebruikt voor de creatie van de melding.
meldingstype Meldingstype 1 .. * De relatie tussen een Melding en een Meldingstype dat gebruikt worden om de melding te categoriseren.
samenvatting String 0 .. 1 Een samenvatting van de Melding. Deze eigenschap kan gebruikt worden als een tekstveld om de Melding en de informatie die eraan gekoppeld is samen te vatten en de behandeling ervan te vergemakkelijken.
titel String 1 .. 1 De titel van een Melding.
url String 0 .. 1 Een doorverwijzing naar een webpagina over deze Melding. Deze eigenschap kan gebruikt worden om een gebruiker door te verwijzen naar een applicatie waar de actuele status van de Melding kan geraadpleegd worden.
heeftStatus Status 1 .. * De Status verbonden aan de Melding.
heeftParticiperende Participatie .. De relatie tussen een Melding en een Agent die een rol speelt in de Melding.
heeftBody MeldingBody 0 .. * Het object van deze associatie is een resource die de body van een Melding beschrijft (MeldingBody).
heeftDoelwit Meldingsobject 0 .. * De relatie tussen een Melding en het doelwit, een Meldingsobject waarop de melding slaat.
Voorbeeld JSON-LD: 
"@id": "http://data.vlaanderen.be/id/melding/1234",
"@type": "Melding",
"meldingsapplicatie": "GRBMS Webloket",
"meldingsorganisatie": "Informatie Vlaanderen",
"meldingstype": "onvolledige gegevens",
"samenvatting": "verbouwing hoofdgebouw | test OSLO²"

Meldingsobject

Het Meldingsobject identificeert een resource, een specifiek deel van een resource,een bepaalde representatie van een resource of een combinatie hiervan welke gebruikt wordt binnen een Melding.

Voor deze entiteit zijn de volgende attributen gedefinieerd: attribuut, heeftBron, heeftStatus, gerelateerdeBody, heeftToestand.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
attribuut MeldingsobjectAttribuut 1 .. 1 Refereert naar een Eigenschap die deel uitmaakt van het Meldingsobject.
heeftBron Dataset 1 .. 1 De bron (Dataset) waar het Meldingsobject werd uit afgeleid. Dit attribuut is relevant in het kader van meldingen op concrete informatievelden zoals foutieve of ontbrekende gegevens (terugmeldingen).
heeftStatus Status 1 .. * De Status verbonden aan het Meldingsobject. Wanneer voor de melding meerdere Meldingsobjecten werden meegegeven die elk door andere Agents worden afgehandeld laat deze eigenschap toe om per Meldingsobject een status mee te geven. De Status van de Melding kan pas behandeld zijn als alle Meldingsobjecten behandeld werden.
gerelateerdeBody MeldingBody 1 .. 1 De body van een Melding die betrekking heeft op het Meldingsobject. Indien een Melding meerdere Meldingsobjecten bevat biedt deze eigenschap de mogelijkheid om te beschrijven welke MeldingBody relevant is voor de behandeling van het Meldingsobject.
heeftToestand TijdToestand 1 .. 1 De relatie tussen een Meldingsobject en een TijdToestand.
werd toegewezen aan Agent 0 .. 1 Het toewijzen van een entiteit aan een Agent.
Voorbeeld JSON-LD: 
"@id": "http://data.vlaanderen.be/id/melding/1234",
"@type": "Melding",

      }
    ]
    "gerelateerdeBody": {

Participatie

De klasse Participatie wordt gebruikt om informatie weer te geven met betrekking tot de rol van een Agent in de creatie of behandeling van een Melding.

Voor deze entiteit zijn de volgende attributen gedefinieerd: beschrijving, rol.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
beschrijving String 1 .. 1 Beschrijving van de Participatie.
rol ParticipatieRol 1 .. * De rol die een participerend Agent heeft met betrekking tot de Melding.
Voorbeeld JSON-LD: 
"@id": "http://data.vlaanderen.be/id/melding/1234",
"@type": "Melding",
"heeftParticiperende": [
  {
    "@id": "https://data.vlaanderen.be/id/persoon/00000014182",
    "@type": "Persoon",
    "rol": "Indiener"
  }
]

Status

De Status van een Melding in het verwerkingsproces op een bepaald tijdstip.

Voor deze entiteit zijn de volgende attributen gedefinieerd: beschrijving, statusdatum, statusType, statuswijzigingDoor.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
beschrijving String 1 .. 1 Een beschrijving van de Status.
statusdatum DateTime 1 .. 1 De datum van een Status.
statusType StatusType 1 .. 1 Het type van de Status.
statuswijzigingDoor Agent 1 .. 1 De Agent die deze status heeft geactiveerd.
Voorbeeld JSON-LD: 
"@id": "http://data.vlaanderen.be/id/melding/1234",
"@type": "Melding",
"heeftStatus": [
  {
    "beschrijving": "in behandeling (vergrendeld)",
    "statusdatum": "2017-04-20",
    "statusType": "4",
    "statuswijzigingDoor": "http://data.vlaanderen.be/id/publiekeorganisatie/OVO002949",
    "statuswijzigingObject": "http://data.vlaanderen.be/id/gebouw/1234",
  }
]

TijdToestand

Een TijdToestand legt de datum en het tijdstip vast waarop het Meldingsobject dient geïnterpreteerd te worden.

Voor deze entiteit zijn de volgende attributen gedefinieerd: datumVaststelling.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
datumVaststelling DateTime 1 .. 1 De datum en het tijdstip waarop het Meldingsobject moet geïnterpreteerd worden als zijnde van toepassing op de Melding.
Voorbeeld JSON-LD: 
"@id": "http://data.vlaanderen.be/id/melding/1234",
"@type": "Melding",
"heeftDoelwit": [
  {
    "@id": "http://data.vlaanderen.be/id/gebouw/1234",
    "@type": "Meldingsobject",
    "heeftToestand": {
      "datumVaststelling": "2017-03-20 00:00:00.000"
    }
  }
]

Datatypes

Geometrie

Het datatype Geometrie dient voor het identificeren van een locatie als een punt, lijn, polygoon, etc. door het gebruik van coördinaten in een referentiesysteem (bv. ETRS89)

Voor dit datatype zijn de volgende attributen gedefinieerd: gml, wkt.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
gml GetypeerdeString 0 .. 1 Geography Markup Language foor het beschrijven van geografische objecten. Gebruik http://www.opengis.net/ont/geosparql#gmlLiteral als datatype.
wkt GetypeerdeString 0 .. 1 Well Known Text notatie voor vector-geometrie objecten. Gebruik http://www.opengis.net/ont/geosparql#wktLiteral als datatype.
Voorbeeld JSON-LD: 
"geometrie": {
  "wkt": "POLYGON((199366.326445378 173495.694970526,199374.355501384 173489.410298523,199364.348141372 173477.521786515,199356.552621372 173484.083834518,199366.326445378 173495.694970526))"
}

MeldingBody

De body van een Melding, deze omvat relevante informatie voor de verdere afhandeling van een melding.

Voor dit datatype zijn de volgende attributen gedefinieerd: extraInformatie, geometrie, waarde.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
extraInformatie URI 1 .. 1 Identificeert een resource die aan de MeldingsBody verbonden is en mogelijks speciale verwerking vraagt. Via deze eigenschap is het mogelijk om bv. Foto's of PDF documenten te koppelen aan een Melding.
geometrie Geometrie 1 .. 1 De relatie tussen een MeldingBody en een geometrie. Wanneer een Agent een Melding aanmaakt kan deze bv. een bounding box tekenen op een map. De geometrie hiervan kan weegegeven worden via de eigenschap geometrie van MeldingBody.
waarde String 1 .. 1 Een waarde (tekststring) voor de MeldingBody. Deze eigenschap kan gebruikt worden om extra beschrijvende tekst mee te geven over het Meldingsobject.
Voorbeeld JSON-LD: 
"@id": "http://data.vlaanderen.be/id/melding/1234",
"@type": "Melding",
"heeftBody": [
 {
  "extraInformatie": "http://example.org/photo.jpeg",
  "geometrie": {
    "wkt": "POLYGON((199366.326445378 173495.694970526,199374.355501384 173489.410298523,199364.348141372 173477.521786515,199356.552621372 173484.083834518,199366.326445378 173495.694970526))"
  },
  "waarde": "bouw, verbouwing of sloping van een gebouw"
 }
]

huidigeWaarde String 0 .. 1 De huidige waarde van een Eigenschap.
hvoorgesteldeWaarde String 0 .. 1 De voorgestelde waarde van een Eigenschap.

    }
  ]
}
