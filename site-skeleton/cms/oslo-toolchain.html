The OSLO-Toolchain enables the potential for supporting multilingual data specifications.

The Core Vocabularies are used by several public administrations to attain a minimum level of semantic interoperability for e-Government systems. Core Vocabularies are simplified, re-usable and extensible data models that capture the fundamental characteristics of an entity in a context-neutral fashion. Public administrations can use and extend the Core Vocabularies in different contexts:


  • Development of new systems
  • Information exchange between systems
  • Data integration
  • Open data publishing


But most of the MS publish these Core Vocabularies and their semantic assets in their own (national) language.


â†’ There is the need for an integrated European Knowledge Graph that can be queried by both machines and people with no language barriers.


We have thousands of objects linked to each other, linked to definitions and linked to international standards. Managing the span of control is a challenge in this growing community. In Flanders, we have therefore developed an open source software tool.


Our open source tool monitors what is happening in the working groups, in line with the governance process and it archives it in a version control system so all the steps in managing the knowledge graph are transparent and traceable to all our stakeholders via the website.


When for example a candidate standard becomes a recognized standard, the toolchain will automatically generate all, in the machine-readable information on our website data.vlaanderen.be , including all the specifications and all the assets needed for the API.


This means that the complete management of the knowledge graph runs entirely through automated processes. And now we have also integrated multilingual improvements in the production branches. This allows for all the specifications to be automatically translated in the preferred language.


When other MS also integrate this toolchain which supports the generation of multilingual vocabularies, we can go towards one integrated European Knowledge Graph where the different vocabularies can be queried in an easy way by both machines and people.
